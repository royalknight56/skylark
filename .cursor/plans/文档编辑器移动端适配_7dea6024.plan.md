# 文档编辑器移动端样式兼容

## 现状问题

当前项目完全按照桌面端设计，**零** `@media` 查询，文档编辑器页面在移动端存在以下问题：

- 侧栏 (`Sidebar.tsx`) 固定 `w-16`，在小屏上占据宝贵空间且无法折叠
- 工作区布局 (`layout.tsx`) 为 `flex h-screen w-screen overflow-hidden`，无断点响应
- 编辑器顶栏 (`DocEditor.tsx`) `px-6` 内边距过大，按钮文字在窄屏会溢出
- 工具栏 (`EditorToolbar.tsx`) 按钮密集，仅靠 `overflow-x-auto` 横向滚动体验差
- 编辑内容区 `px-12 py-8` + `max-w-3xl` 在手机上留白过多
- 表格浮动菜单 (`TableMenu.tsx`) 单行排列 15+ 个按钮，手机上严重溢出
- 表格单元格 `min-width: 100px` 导致表格无法在窄屏自适应
- 分栏布局 (`.columns`) 在手机上不应并排显示

## 适配策略

使用 Tailwind 响应式类 + `styles.css` 中的 `@media` 查询双管齐下，断点以 **768px** (`md`) 为主分界：
- `< 768px`: 移动端布局（侧栏底部 tab、工具栏折叠、表格菜单分行）
- `>= 768px`: 保持现有桌面端布局不变

## 修改文件一览

### 1. 全局 viewport 元数据 -- [src/app/layout.tsx](src/app/layout.tsx)

导出 `viewport` 对象，确保移动端正确缩放：

```typescript
export const viewport = {
  width: "device-width",
  initialScale: 1,
  maximumScale: 1,
};
```

### 2. 工作区布局 -- [src/app/(workspace)/layout.tsx](src/app/(workspace)/layout.tsx)

- 桌面端 (`md:`) 保持现有 `flex` 横向布局 + 左侧 `Sidebar`
- 移动端改为 `flex-col`，`Sidebar` 改到底部显示（底部 Tab 栏）
- `main` 占满剩余高度

关键改动：

```tsx
<div className="flex flex-col md:flex-row h-screen w-screen overflow-hidden">
  <Sidebar />
  <main className="flex-1 flex overflow-hidden relative">
    ...
  </main>
</div>
```

### 3. 侧栏导航 -- [src/components/layout/Sidebar.tsx](src/components/layout/Sidebar.tsx)

- **移动端**：变为底部固定 Tab 栏（`fixed bottom-0`），仅显示图标，水平排列5个主要导航项
- **桌面端** (`md:`)：保持现有的左侧竖向侧栏 `w-16`
- 企业切换 / 设置 / 头像在移动端隐藏或收入更多菜单
- 使用 Tailwind 断点类实现，无需 JavaScript 判断

布局切换思路：
```
移动端: aside 固定在底部 → h-14 w-full flex-row
桌面端: aside 固定在左侧 → w-16 h-screen flex-col
```

### 4. 文档编辑器 -- [src/components/docs/DocEditor.tsx](src/components/docs/DocEditor.tsx)

**顶栏适配**：
- `px-6` 改为 `px-3 md:px-6`
- 标题 input 字号：`text-sm md:text-base`
- "保存于" 时间戳在移动端隐藏：`hidden md:flex`
- 按钮组在移动端紧凑排列，"保存"按钮文字隐藏仅显示图标

**编辑区适配**：
- `px-12 py-8` 改为 `px-4 py-4 md:px-12 md:py-8`
- `max-w-3xl` 改为 `max-w-full md:max-w-3xl`

### 5. 编辑器工具栏 -- [src/components/docs/editor/EditorToolbar.tsx](src/components/docs/editor/EditorToolbar.tsx)

- 保持 `overflow-x-auto` 横向滚动
- 在移动端将按钮分为两行或使用更紧凑的间距：`gap-0 md:gap-0.5`
- 部分低频按钮在移动端隐藏（通过 `hidden md:flex` ），保留核心格式按钮
- 建议隐藏的低频按钮：缩进/减少缩进、分栏、视频、Mermaid、进度条、阅读模式（顶栏已有）
- 减小按钮尺寸：`w-7 h-7 md:w-8 md:h-8`

### 6. 表格浮动菜单 -- [src/components/docs/editor/TableMenu.tsx](src/components/docs/editor/TableMenu.tsx)

这是移动端适配的重点难点，当前 15+ 按钮单行排列必然溢出。

**方案：多行 flex-wrap 布局**

- 外层容器添加 `flex-wrap`，让按钮自然换行
- 移动端最大宽度限制为屏幕宽度 - 边距：`max-w-[calc(100vw-2rem)] md:max-w-none`
- 分隔符 `Sep` 在移动端不显示（`hidden md:block`）
- 颜色选择器弹窗位置自适应（避免超出屏幕）

```tsx
<div className="flex items-center flex-wrap gap-0.5 max-w-[calc(100vw-2rem)] md:max-w-none ...">
```

### 7. 编辑器样式 -- [src/components/docs/editor/styles.css](src/components/docs/editor/styles.css)

添加 `@media (max-width: 767px)` 媒体查询：

```css
@media (max-width: 767px) {
  /* 表格横向滚动包装 */
  .tiptap-editor table {
    display: block;
    overflow-x: auto;
    -webkit-overflow-scrolling: touch;
  }
  
  /* 缩小单元格最小宽度 */
  .tiptap-editor th,
  .tiptap-editor td {
    min-width: 80px;
    padding: 0.375rem 0.5rem;
    font-size: 0.875rem;
  }
  
  /* 分栏布局在手机上竖向堆叠 */
  .tiptap-editor .columns {
    flex-direction: column;
  }
  
  /* 代码块字号缩小 */
  .tiptap-editor pre {
    font-size: 0.8125rem;
    padding: 0.75rem 1rem;
  }
  
  /* 标题字号缩小 */
  .tiptap-editor h1 { font-size: 1.5rem; }
  .tiptap-editor h2 { font-size: 1.25rem; }
  .tiptap-editor h3 { font-size: 1.125rem; }

  /* 编辑器最小高度减小 */
  .tiptap-editor { min-height: 300px; }
}
```

### 8. 全局滚动条 -- [src/app/globals.css](src/app/globals.css)

移动端隐藏自定义滚动条（移动端原生滚动更自然）：

```css
@media (max-width: 767px) {
  ::-webkit-scrollbar { display: none; }
  * { -webkit-overflow-scrolling: touch; }
}
```

## 不修改的部分

- `BubbleToolbar.tsx` -- 已经足够紧凑，浮动在选中文本附近，无需改动
- `SlashMenu.tsx` / `SearchReplace.tsx` / `LinkModal.tsx` -- 弹窗/模态框通常已居中显示，优先级低
- `TOCSidebar` -- 目录侧栏在移动端可通过按钮切换已足够
